7. Introduction

7.1 Purpose of this Document

This SRS documents the requirements for the Job Portal web application. It defines the functional and non-functional requirements, system architecture expectations, database schema, API endpoints, validation rules, security expectations, test cases and risk considerations.

7.2 Intended Audience

Project Supervisor, Developers, Testers, Maintainers.

8. Purpose

The Job Portal provides an online platform for job discovery and application management including profile management, company registration, job posting, and application workflows.

9. Scope

Web-based front-end (React + Vite), RESTful backend (Node/Express), MongoDB, Cloudinary for file storage.

10. Product Features

  • Authentication & Authorization
  • Company Management
  • Job Management
  • Application Management
  • Profile Management
  • Search & Filtering

11. Functional Requirements (selected)

IDRequirement
FR-1User Registration: create user with hashed password.
FR-2User Login: authenticate and set JWT cookie.
FR-8Post Job: recruiter posts new job.
FR-12Apply Job: create Application, prevent duplicates.

12. Database Design

Collections: users, companies, jobs, applications. See repository models for exact schema definitions.

13. API Summary

Key endpoints include /api/v1/user/register, /api/v1/user/login, /api/v1/job/post, /api/v1/application/apply/:id (should be POST).

14. Security

Recommendations: set cookie httpOnly & secure; validate uploads; rate-limit login; add owner checks.

15. Testing

Suggested tools: Jest, Supertest, React Testing Library, Cypress. Use separate test DB and mock Cloudinary.

16. Diagrams (textual)

Textual descriptions of System Architecture, ER Diagram, Use Case, DFD Level 0/1, Activity, Sequence, Component and Deployment diagrams are included in the full markdown file.

17. Screenshots Placeholders

Placeholders for Home, Login, Signup, Profile, Job Listings, Job Description, Apply Confirmation, Recruiter Dashboard, Company Management, Post Job Form, Applicants Management.

18. Risk Analysis

Key risks include cookie configuration, memory usage with file uploads, authorization checks, and malicious uploads. Mitigations are documented in the SRS markdown.
